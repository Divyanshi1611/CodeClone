About the Project

What is Code Clone Detection?

Clone detection is a static analysis technique used to identify syntactic redundancy in software development artifacts, such as code. It involves identifying parts of the artifact that appear more than once, known as clones. These clones can be exact copies or have some variations, such as changes in identifiers or added/removed lines of code. Clone detection helps identify and address code duplication, which can lead to larger codebases, decreased readability, increased review effort, and potential introduction of faults. It can be applied to various types of texts, not just source code.

Types of Code Clones

Clone Type Description
Type-1 or Exact Copy These type of clones look like original code with variation of comments and blank spaces. They can be detected by use of text based, token based or any simple clone detection technique. Tools based on algorithms like KMP, SDD algorithm can detect these clones by string matching.
Type-2 or Renamed Clone Variations in name of literals, variables, keywords creates renamed or type 2 clones. The use of token or metric based techniques will be able to detect these kinds of clones in a code. CLAN, Columbus, MCD-finder are some of the tools used by developers to detect renamed clones.
Type-3 or Near miss clone They are created from base code by statement modification, addition or deletion. Tools like ClonedDr, Clone Digger based on tree technique can be used to detect near miss clones. The AST sub tress is compared with each other in tree based techniques.
Type-4 or Semantic clone The program coding or syntax is different but the behaviour or function of the clone remains same in semantic clones. Tools like Duplex, Scorpio based on graph technique are proved to be fruitful in detecting these types of clones.
